Khảo sát API Quản lý Thư viện (RAML) version v1

http://localhost:8000

Tài liệu này thể hiện API thiết kế dưới định dạng RAML (RESTful API Modeling Language). Đặc trưng nổi bật ở định dạng này là giảm lặp code (DRY) thông qua việc hệ thống mô hình cây endpoints rất trực quan và cung cấp traits cùng types.

/books

Quản lý thông tin sách trong kho thư viện.

/books get post

get

Lấy danh sách toàn bộ sách trong hệ thống.

post

Thêm mới sách vào hệ thống API.

get /books

Lấy danh sách toàn bộ sách trong hệ thống.

  • Request
  • Response

Query Parameters

  • status: (one of available, borrowed)

    Tham số lọc lấy những sách có trạng thái tương ứng.

HTTP status code 200

Body

Media type: application/json

Type: array of object

Items: Book

  • id: required(integer)

    Example:

    1
  • title: required(string)

    Example:

    Sapiens: Lược Sử Loài Người
  • author: required(string)

    Example:

    Yuval Noah Harari
  • status: required(one of available, borrowed)

    Example:

    available
